# Visão de Produto — OdontoOS

---

## O que é o OdontoOS

OdontoOS é um sistema operacional de consultório odontológico.

Não é um ERP odontológico tradicional.
Não é apenas uma agenda.
Não é apenas um prontuário.

É uma plataforma operacional para pequenos consultórios
e dentistas autônomos, com foco em organização, velocidade,
clareza e profissionalismo.

---

## Identidade central

OdontoOS = camada digital de profissionalismo.

Um dentista pequeno ou clínica enxuta deve parecer mais
organizada, estruturada e confiável no atendimento ao paciente.

---

## Lógica principal de uso
buscar → selecionar → executar

Exemplo:
A recepcionista abre o sistema, digita "lim",
encontra "Limpeza", seleciona o paciente,
agenda ou gera orçamento rapidamente.

---

## Princípios invioláveis

1. Busca rápida como centro do sistema
2. Procedimentos favoritos no topo
3. Ações rápidas com poucos cliques
4. Interface limpa — sem menus complexos
5. Sensação de controle e fluidez
6. Velocidade acima de excesso de funções
7. Linguagem simples — sem termos técnicos
8. Cada ação deve responder: isso ajuda alguém
   a trabalhar mais rápido no consultório?
   Se não — não entra agora.

---

## MVP — Primeira versão

O MVP deve conter apenas:

1. Login com autenticação real
2. Dashboard operacional com dados reais
3. Cadastro e busca de pacientes
4. Agenda visual
5. Lista de procedimentos com busca e favoritos
6. Criação de orçamento
7. Registro básico de pagamento
8. Histórico simples do paciente

Fora do MVP — não entra agora:
- IA
- Relatórios avançados
- Prontuário complexo
- Odontograma
- WhatsApp
- Automações
- App do paciente
- Marketing

---

## Faseação do produto

### Fase 1 — Base operacional
- Autenticação real
- Dashboard com dados reais
- Navegação sem rotas quebradas
- Estrutura visual consolidada

### Fase 2 — Pacientes e agenda
- Cadastro completo de pacientes
- Busca rápida por nome
- Agenda visual funcional
- Histórico básico

### Fase 3 — Procedimentos e orçamento
- Lista de procedimentos
- Busca por procedimento
- Favoritos por dentista
- Geração de orçamento

### Fase 4 — Financeiro básico
- Registro de pagamentos
- Status de orçamentos
- Controle simples de recebimentos

### Fase 5 — Prontuário
- Registro de atendimentos
- Histórico clínico básico
- Evolução por consulta

### Fase 6 — Inteligência operacional
- Automações simples
- Lembretes
- Confirmações
- Relatórios básicos

---

## Regras de UX

1. Toda tela deve ter uma ação principal clara
2. Busca sempre visível e acessível
3. Favoritos sempre no topo
4. Nenhuma ação deve exigir mais de 3 cliques
5. Erros devem ser visíveis e compreensíveis
6. Feedback imediato para toda ação do usuário
7. Mobile deve funcionar tão bem quanto desktop

---

## Regras de desenvolvimento

1. Não criar telas sem seguir esta visão
2. Não duplicar componentes
3. Não misturar lógica com visual sem necessidade
4. Não criar funcionalidades fora do MVP atual
5. Sempre entregar código completo
6. Sempre explicar onde colocar cada arquivo
7. Manter modularidade
8. Atualizar CONTEXT.md ao concluir cada feature
9. Registrar decisões em DECISIONS.md

---

## Pergunta de validação para cada decisão

"Isso ajuda alguém a trabalhar mais rápido
no consultório odontológico?"

Se a resposta for não — não entra agora.
